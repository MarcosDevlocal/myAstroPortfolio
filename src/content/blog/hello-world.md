This blog runs on Markdown inside Astro, which means adding a post is as simple as creating a new file in `src/content/blog/` with frontmatter and hitting deploy.

I plan to write about:

- Backend craft — APIs, data modeling, and the boring parts that keep software alive
- Real projects — what worked, what broke, what I would redo
- Konari — how we use technology in service of people, not the other way around

If something here helps you ship calmer software, that is already a win.
